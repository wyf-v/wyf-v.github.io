搭建Hexo博客站点
发表于2020-09-01|更新于2020-09-06
|阅读量:

我有一个Vps的小主机，之前使用的是PHP平台的wordpress博客站，由于性能问题，选择使用Hexo搭建网站，以下是搭建过程

Hexo是基于Node.js开发的博客框架。安装完成后其工作流程为:

graph LR
创建md文档-->用MD编写博客-->生成网站-->上传网站

并且由于生成的是静态网站，在弱鸡的vps上也能快速地打开访问。

我可以在我的笔记本上生成网站，然后再将生成结果上传到服务器，这能进一步的降低服务器的压力。以下是搭建过程

我的配置是MacBook+CentOS系统的服务器。先是在本地安装Hexo

本地安装Hexo

参考官网文档。安装Hexo需要Node.js 支持

安装Node.js

Node.js可以通过下载安装包安装，也可以通过包管理器方式安装。

我是使用Homebrew包管理器:

检查安装情况

如果没有Git还需要安装Git.

安装Hexo

npm是Node.js的包管理器，使用npm安装Hexo

-g ：全局安装

创建Hexo

Hexo项目可以创建多个，可以通过命令hexo init "项目名字"创建一个项目

tree -L 指定列出目录深度

配置服务器并上传到服务器

Hexo自带推送功能，我选择以Git的方式推送到服务器。git推送是以ssh链接的方式推送。ssh可以使用证书登录，不用输入用户密码。

在服务器配置免密登录：

创建并配置git仓库

如果没安装Git，安装Git:yum install git

以Hexo用户登录服务器后：

修改如下

git init –bare 创建一个裸仓库，没有工作区。可以通过hooks将源码和仓库分离

将服务器的网站根目录设置为/home/hexo/blog/html

配置本地hexo的上传

打开本地Hexo项目的配置文件_config.yml 修改deploy:的内容

然后执行命令hexo d

hexo g -d 等于执行hexo g后执行hexo d
